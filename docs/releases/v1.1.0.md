# Agent Signal Bar v1.1.0

## English

### Highlights

- Major UI refresh for the settings window and menu bar panel, including a cleaner Liquid Glass visual system.
- Codex Desktop monitoring is now treated as the default path for Codex Desktop users, so normal Desktop use does not require manually installing hooks.
- The menu bar light, the menu bar panel, and the Activity settings page now share the same live display model so their visible status stays aligned.
- README visuals were fully refreshed with current screenshots for the menu bar panel, Activity page, Liquid Glass comparison, and animated light effects.

### Changes

- Reworked the settings window into a simpler five-page structure: Activity, General, Connect, Advanced, and About.
- Moved visual style, status-menu mode, light-effect customization, and lower-level controls into Advanced so the main settings pages stay focused.
- Updated the settings tab bar, activity cards, session rows, recent-event list, and dropdown controls to match the current Liquid Glass design language.
- Added a solid-vs-Liquid-Glass comparison to the README using real screen-region screenshots with desktop background participation.
- Regenerated README status-light GIFs from the real status bar icon renderer.
- Refreshed English and Chinese README screenshots for the current app UI.
- Clarified Codex Desktop automatic monitoring versus optional Codex CLI/IDE hooks in the README and setup docs.
- Improved Codex Desktop log monitoring so startup does not replay old history as fresh work.
- Improved active-session filtering so the Activity page can show all visible agents while the signal light can still focus on a selected light agent.
- Improved release packaging and diagnostics scripts, including safer uninstall state cleanup and refreshed release verification paths.
- Kept internal marketing assets ignored and out of the public release package.
- Bumped app and release metadata to 1.1.0.

### Fixes

- Fixed cases where the status bar light, menu bar panel, and settings Activity page could disagree about whether Codex was idle or working.
- Fixed delayed or stale Codex events overriding newer session state.
- Fixed late session-end events clearing paused/off state.
- Fixed old Codex Desktop log entries being treated as new activity when the app starts.
- Fixed launch-at-login state reporting so failed setup does not appear enabled.
- Fixed dropdown, segmented-control, action-button, and recent-event alignment issues in the settings window.
- Fixed menu bar panel layout inconsistencies around quick actions, current events, and recent events.
- Fixed README images that were still showing older UI or old Liquid Glass wording.

### Release verification

- GitHub Actions builds the app, runs Swift tests, packages the release, and verifies SHA-256 checksums.
- Local release packaging uses `script/package_release.sh`.
- Checksums are published in [AgentSignalLight-SHA256SUMS.txt](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.1.0/AgentSignalLight-SHA256SUMS.txt).
- Release metadata is published in [AgentSignalLight-release-manifest.json](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.1.0/AgentSignalLight-release-manifest.json).
- This is a local/self-use macOS build. It is ad-hoc signed and not notarized.

## 简体中文

### 亮点

- 设置窗口和状态栏小菜单进行大幅 UI 更新，加入更统一、更干净的液态玻璃视觉系统。
- Codex Desktop 自动监控成为 Codex Desktop 用户的默认路径，普通 Desktop 使用不需要手动安装 Hook。
- 状态栏灯效、状态栏小菜单和设置里的运行页现在使用同一套实时显示模型，避免可见状态不同步。
- README 图片全面更新为当前版本截图，包括状态栏小菜单、运行页、液态玻璃对比图和动态灯效预览。

### 改进

- 设置窗口重构为更简单的五个页面：运行、通用、连接、高级、关于。
- 将状态栏样式、状态栏菜单模式、灯效自定义和底层控制项移动到高级页，让主设置页面更聚焦。
- 更新设置菜单栏、运行卡片、当前会话、最近事件、下拉框和分段控件，使其更接近当前液态玻璃风格。
- README 新增无液态玻璃 / 默认液态玻璃对比图，并使用真实区域截图，让桌面背景参与到窗口材质中。
- 使用真实状态栏图标渲染器重新生成 README 里的状态灯动图。
- 更新中英文 README 中所有当前展示用截图。
- 在 README 和接入文档中进一步说明 Codex Desktop 自动监控与可选 Codex CLI/IDE Hook 的区别。
- 优化 Codex Desktop 日志监控，App 启动时不会把旧日志重放成新的工作状态。
- 优化当前会话过滤逻辑：运行页可以展示所有可见 Agent，状态栏灯效仍可聚焦某个选中的灯效 Agent。
- 优化发布打包和诊断脚本，包括更安全的卸载状态清理和更新后的发布验证路径。
- 继续保持内部 marketing 素材被忽略，不进入公开仓库和发布包。
- 将 App 和发布元数据更新到 1.1.0。

### 修复

- 修复状态栏灯效、状态栏小菜单和设置运行页在 Codex 空闲 / 工作判断上可能不同步的问题。
- 修复延迟到达或过期的 Codex 事件覆盖较新 session 状态的问题。
- 修复迟到的 session_end 事件可能清掉暂停 / 关闭状态的问题。
- 修复 App 启动时旧 Codex Desktop 日志被当作新活动的问题。
- 修复开机自启动状态显示不准确的问题，避免设置失败但 UI 看起来已经开启。
- 修复设置窗口中下拉框、分段控件、操作按钮和最近事件列表的对齐问题。
- 修复状态栏小菜单快捷按钮、当前事件和最近事件的布局不一致问题。
- 修复 README 中仍然显示旧 UI 或旧液态玻璃文案的图片。
