# Agent Signal Bar v1.2.0

## English

### Highlights

- Added source-aware Codex monitoring for Codex Desktop, Codex CLI, Codex VS Code, Codex Xcode, and Codex IDEA.
- Codex monitoring now works as a hook-free default path by reading local Codex session logs and platform activity.
- Added a multi-select Light Agent picker so the menu bar light can follow one or more supported agent sources.
- Improved the menu bar panel so its live sessions and recent events stay aligned with the Activity page.
- Hardened release packaging for public GitHub downloads.

### Changes

- Added platform presence detection for supported Codex entrypoints.
- Improved activity source labeling so Codex Desktop, terminal Codex, and IDE Codex are easier to tell apart.
- Updated the Connect page copy to describe Codex monitoring as a supported built-in path, with hooks kept as optional compatibility.
- Added uninstall actions for optional Codex and Claude hooks.
- Kept Claude support available through Claude Desktop monitoring and optional Claude hooks, while continuing to mark Claude as not fully field-tested.
- Increased recent-event retention to support larger activity histories.
- Updated public release assets to use the product-facing `AgentSignalBar` download names.

### Fixes

- Fixed a status bug where completed Codex activity could be revived by delayed old active events and appear as thinking or working again.
- Fixed stale source attribution where VS Code, Xcode, or IDEA hints could be overridden by an older cached desktop source.
- Fixed Codex log polling continuing when Codex monitoring was disabled but Claude monitoring remained enabled.
- Fixed hook payload parsing so tool arguments or user input fields named `session_id`, `source`, or `error` no longer cause false sessions or false red-light states.
- Fixed platform process scanning so large `ps` output cannot block detection and make CLI / IDE presence look missing.
- Fixed current-session and recent-event duplication when fallback activity is generated from recent events.
- Fixed hook uninstall behavior so project-level Codex hooks are removed along with user-level hooks.
- Fixed release zip contents to include root metadata files and exclude internal marketing / social / video folders.
- Fixed notarization metadata refresh so custom DMG paths update checksums and release manifests correctly.

## 简体中文

### 亮点

- 新增 Codex 多入口识别：Codex Desktop、Codex CLI、Codex VS Code、Codex Xcode、Codex IDEA。
- Codex 默认可以不依赖 Hook，通过本地 Codex session 日志和平台活动识别运行状态。
- 新增可多选的“灯效 Agent”选择器，可以指定状态栏灯效跟随一个或多个支持来源。
- 优化状态栏小菜单，让实时会话和最近事件与设置里的“运行”页面保持一致。
- 强化 GitHub 公开下载包和发布流程。

### 改进

- 新增平台存在检测，用于识别支持的 Codex 入口。
- 优化活动来源显示，让桌面版 Codex、终端 Codex、IDE Codex 更容易区分。
- 更新“连接”页面文案，把 Codex 自动监控描述为内置支持能力，Hook 保留为可选兼容方案。
- 为可选 Codex Hook 和 Claude Hook 增加卸载操作。
- 保留 Claude Desktop 监控和 Claude Hook 支持，同时继续标注 Claude 尚未完成充分实机验证。
- 增加最近事件保留能力，支持更长的活动历史。
- 公开发布产物统一使用面向用户的 `AgentSignalBar` 下载名称。

### 修复

- 修复 Codex 完成后可能被延迟的旧 active 事件复活，重新显示为思考中或工作中的问题。
- 修复 VS Code、Xcode、IDEA 来源提示可能被旧桌面来源缓存覆盖的问题。
- 修复关闭 Codex 监控后，只要 Claude 监控仍开启，Codex 日志轮询仍会运行的问题。
- 修复 Hook payload 解析过宽导致工具参数或用户输入里的 `session_id`、`source`、`error` 被误认为真实会话、来源或失败状态的问题。
- 修复平台进程扫描在 `ps` 输出过大时可能阻塞，导致 CLI / IDE 存在性误判的问题。
- 修复由最近事件生成 fallback 活动时，当前会话和最近事件重复显示的问题。
- 修复卸载脚本只删除用户级 Hook、不删除项目级 Codex Hook 的问题。
- 修复 release zip 缺少根目录元数据文件、以及内部 marketing / social / video 目录可能混入发布包的问题。
- 修复 notarization 使用自定义 DMG 路径时 checksum 和 release manifest 刷新不准确的问题。
