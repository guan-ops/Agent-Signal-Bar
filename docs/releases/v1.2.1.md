# Agent Signal Bar v1.2.1

## English

### Highlights

- Fixed the About page update checker so GitHub API rate limits or HTTP 403 responses no longer make the update check stop immediately.
- Added a public GitHub Releases page fallback for update checks.
- Confirmed automatic monitoring is enabled by default for fresh installs.

### Fixes

- The app now first tries the GitHub Releases API.
- If the API request fails, the app falls back to the public `/releases/latest` page and reads the redirected release tag.
- This keeps the "Check for Updates" button useful even when GitHub API access is temporarily limited.
- Added a regression test that simulates an API HTTP 403 response and verifies that the release-page fallback still detects the latest version.
- Added regression coverage that verifies fresh installs start with Codex and Claude automatic monitoring enabled.

## 简体中文

### 亮点

- 修复“关于”页面里的检查更新功能，GitHub API 限流或返回 HTTP 403 时不再直接失败。
- 为检查更新增加公开 GitHub Releases 页面 fallback。
- 确认新安装时自动监控默认开启。

### 修复

- App 会优先请求 GitHub Releases API。
- 如果 API 请求失败，App 会自动降级到公开的 `/releases/latest` 页面，并通过重定向后的 release tag 判断最新版本。
- 即使 GitHub API 暂时受限，“检查更新”按钮也能继续给出可用结果。
- 新增回归测试，模拟 GitHub API HTTP 403，并验证 release 页面 fallback 仍能识别最新版本。
- 新增回归测试，确认新安装时 Codex 和 Claude 自动监控默认开启。
